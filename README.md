# Vitiscan вҖ” Diagnostic API

REST API for grape leaf disease classification using a fine-tuned CNN model (ResNet18).  
Part of the **Vitiscan MLOps pipeline** вҖ” a certification project demonstrating production-grade MLOps practices.

---

## Overview

The Diagnostic API accepts a grape leaf photo and returns a ranked list of disease predictions
with confidence scores. It uses a ResNet18 model loaded from the MLflow model registry at startup.

**Live endpoint:** `https://mouniat-vitiscanpro-diagno.hf.space`  
**Interactive docs:** `https://mouniat-vitiscanpro-diagno.hf.space/docs`

---

## Disease Classes

The model classifies leaves into **7 categories** (6 diseases + healthy):

| Class key | Common name |
|-----------|-------------|
| `colomerus_vitis` | Erinose |
| `elsinoe_ampelina` | Anthracnose |
| `erysiphe_necator` | Powdery mildew (Oidium) |
| `guignardia_bidwellii` | Black rot |
| `healthy` | Healthy leaf |
| `phaeomoniella_chlamydospora` | Esca |
| `plasmopara_viticola` | Downy mildew (Mildiou) |

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Health check вҖ” returns `{"status": "ok"}` |
| `GET` | `/diseases` | Lists all 7 detectable classes |
| `POST` | `/diagno` | Runs CNN inference on an uploaded leaf image |
| `GET` | `/docs` | Swagger UI (auto-generated by FastAPI) |

### POST /diagno вҖ” Request & Response

**Request:** `multipart/form-data` with a `file` field (JPEG or PNG image).

```bash
curl -X POST "https://mouniat-vitiscanpro-diagno.hf.space/diagno" \
     -F "file=@leaf_photo.jpg"
```

**Response:**
```json
{
  "predictions": [
    {"disease": "plasmopara_viticola", "confidence": 0.923},
    {"disease": "erysiphe_necator",    "confidence": 0.041},
    {"disease": "healthy",             "confidence": 0.018},
    ...
  ],
  "model_version": "vitiscan-resnet18"
}
```

Predictions are always sorted from highest to lowest confidence.  
All 7 classes are returned. Confidence scores sum to 1.0 (softmax output).

## Local Development

### Prerequisites

- Python 3.11
- AWS credentials with read access to `vitiscanpro-bucket`
- Access to the MLflow tracking server

### 1 вҖ” Clone the repository

```bash
git clone https://github.com/VITISCANPRO-Public/Diagnostic_API.git
cd Diagnostic_API
```

### 2 вҖ” Create and activate the conda environment

```bash
# CPU (local development)
conda env create -f environment.yml
conda activate vitiscan_api_diagno

# GPU (EC2 training instance)
conda env create -f env_api_diagno_gpu.yml
conda activate vitiscan_api_diagno
```

### 3 вҖ” Configure environment variables

```bash
cp .env.template .env
# Edit .env and fill in your credentials (see Environment Variables section below)
```

### 4 вҖ” Run locally

```bash
uvicorn app:app --host 127.0.0.1 --port 4000 --reload
```

API docs available at: `http://127.0.0.1:4000/docs`

## Environment Variables

Copy `.env.template` to `.env` and fill in the values below.

| Variable | Description | Example |
|----------|-------------|---------|
| `MLFLOW_TRACKING_URI` | URL of the MLflow tracking server | `https://mouniat-vitiscanpro-hf.hf.space` |
| `MLFLOW_MODEL_URI` | Model URI in MLflow registry вҖ” format: `models:/name/version` | `models:/vitiscan-resnet18@production` |
| `MODEL_ARTIFACT_ROOT` | S3 path to the model artifact folder (used to load `disease-inrae.json`) | `s3://vitiscanpro-bucket/mlflow-artifacts/models/RUN_ID/artifacts` |
| `S3_BUCKET_NAME` | AWS S3 bucket name | `vitiscanpro-bucket` |
| `DATASET_NAME` | Dataset identifier, used to load the correct disease label file | `inrae` |
| `AWS_ACCESS_KEY_ID` | AWS access key | `AKIA...` |
| `AWS_SECRET_ACCESS_KEY` | AWS secret key | `...` |
| `AWS_DEFAULT_REGION` | AWS region | `eu-west-3` |

> **`MLFLOW_MODEL_URI` format:** always use the Registry format `models:/model-name/version`.  
> You can find the model name and version on your MLflow server under **Models вҶ’ vitiscan-resnet18**.

> **`MODEL_ARTIFACT_ROOT`:** find the RUN_ID on your MLflow server вҖ” click on a training run,
> then look at the **Artifacts** tab to see the full S3 path.

## Testing

Tests use a lightweight `MockModel` (uniform logits, no GPU or cloud credentials needed).  
They are automatically triggered by GitHub Actions on every push to `main`.

```bash
# Run tests locally
TESTING=true pytest tests/test_api_integration.py -v
```

The test suite covers:
- Health check endpoint (`/`)
- Disease list endpoint (`/diseases`) вҖ” verifies all 7 INRAE classes are present
- Inference endpoint (`/diagno`) вҖ” verifies response format, sorted confidence scores, sum = 1.0
- Error handling вҖ” missing file (422), non-image file (400/500)

## Deployment

The API is deployed on **HuggingFace Spaces** using a Docker container.  
Deployment is **automated by GitHub Actions** on every push to `main` that passes all tests.

```
git push origin main
    вҶ’ Unit tests          (pytest Model_CNN/tests/)
    вҶ’ Integration tests   (pytest tests/test_api_integration.py вҖ” TESTING=true)
    вҶ’ Deploy to HF Spaces (only if both test jobs pass)
```

The following secrets must be configured in **GitHub repository settings вҶ’ Secrets and variables вҶ’ Actions**:

| Secret name | Description |
|-------------|-------------|
| `HF_TOKEN` | HuggingFace token with write access to the Space |

The following secrets must be configured in **HuggingFace Space settings вҶ’ Repository secrets**:

| Secret name | Description |
|-------------|-------------|
| `MLFLOW_TRACKING_URI` | MLflow server URL |
| `MLFLOW_MODEL_URI` | Model registry URI (`models:/name/version`) |
| `MODEL_ARTIFACT_ROOT` | S3 path to model artifacts |
| `S3_BUCKET_NAME` | S3 bucket name |
| `DATASET_NAME` | `inrae` |
| `AWS_ACCESS_KEY_ID` | AWS access key |
| `AWS_SECRET_ACCESS_KEY` | AWS secret key |
| `AWS_DEFAULT_REGION` | `eu-west-3` |

## One-time Setup вҖ” Upload Disease Labels to S3

Before running the API for the first time in production, upload the disease label dictionary to S3:

```bash
# Make sure your .env is configured with AWS credentials
python scripts/init_s3.py
```

This uploads `disease-inrae.json` to `s3://vitiscanpro-bucket/vitiscan-data/`.
